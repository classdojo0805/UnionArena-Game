# Union Arena 遊戲機制規範 (MECHANICS.md) - v1.4 (Comprehensive Logic)

本文件為 UA-Connect 專案的核心規範 (Source of Truth)，所有數據解析與遊戲邏輯必須遵循此定義。

## 1. 效果發動時機 (Activation Timing)

| 代碼 | 官方術語 | 說明 | 預設回合上下文 |
| :--- | :--- | :--- | :--- |
| `on_play` | 【登場時】 | 進入場景區時觸發。**注意：因 Raid Trigger 可能在對手回合發動，此時機預設為 `both_turns`**。 | `both_turns` |
| `main_act` | 【主啟動】 | 玩家主要階段手動發動。 | `own_turn` |
| `on_attack` | 【攻擊時】 | 宣告攻擊時觸發。 | `own_turn` |
| `on_block` | 【阻擋時】 | 宣告阻擋時觸發。 | `opponent_turn` |
| `on_retire` | 【退場時】 | 從場地送往退場區時觸發。 | `both_turns` |
| `on_end_turn` | 【回合結束時】 | 雙方回合結束時判定。 | `both_turns` |
| `start_of_turn` | 【回合開始時】 | 玩家重置階段後的時點。 | `own_turn` |
| `constant` | 【永續】 | 只要此卡片在場景區，效果即持續生效。 | `both_turns` |

### 回合上下文 (Turn Context)
- `own_turn`: 僅在自己回合生效。
- `opponent_turn`: 僅在對手回合生效。
- `both_turns`: 雙方回合皆可生效。

## 2. 生命觸發 (Trigger Effect)

當卡片從生命區 (Life Area) 被翻開時觸發的效果。

| 觸發類型 | 精確效果定義 (v.231117) |
| :--- | :--- |
| `draw` | 玩家從牌庫抽取 1 張卡片。 |
| `get` | 將此卡片加入手牌。 |
| `active` | 選擇 1 張我方角色卡轉為「激活 (Active)」狀態，且**該角色在本回合中 BP +3000**。 |
| `final` | **若此卡片是生命區的最後一張卡片**，則從牌庫頂部將 1 張卡片放入生命區 (Life Recovery)。 |
| `color` | **藍色**：BP3500以下對手角色彈手；**紅色**：BP2500以下對手角色退場；**紫色**：棄牌區 2能1AP 角色激活登場；**綠色**：手牌 2能1AP 角色激活登場；**黃色**：對手一張牌休息且下次無法激活。 |
| `special` | 選擇對手 1 張角色卡並將其退場 |
| `raid` | **[突襲] 判定**: 若滿足能量需求與特定基本卡名稱，玩家可選擇執行 [Raid] 登場；若不滿足或玩家不選擇，則將此卡片加入手牌。 |

## 3. 特殊標籤與關鍵字 (Keywords)

- **[Impact X]**: 戰鬥勝利時，額外對對手生命造成 X 點傷害。
- **[Snipe]**: 攻擊時可選擇對手能量區的角色作為攻擊對象。
- **[Damage X]**: 攻擊命中生命區時造成的傷害點數。
- **`must_be_blocked`**: **必擋**。此卡攻擊時，若對手有處於激活狀態且可阻擋的角色，對手**必須**宣告阻擋。
- **`step`**: **滑步**。移動回合時，可將此卡從戰鬥區移動到能量區。
- **`null_impact`**: **衝擊無效**。此角色受到的 [Impact] 傷害無效。
- **`extra_attack_2`**: **兩次攻擊**。此角色在攻擊後會激活。
- **`extra_block_2`**: **兩次阻擋**。此角色在阻擋後會激活。
- **`impact_plus_x`**: **衝擊 +1**。賦予原本的 [Impact] 數值再加 X。
- **`damage_plus_x`**: **傷害 +1**。命中生命區時的傷害點數增加 X。

## 4. 進階控制與狀態 (Control & States)

- **`skip_refresh_active`**: **下一次不會被激活**。下次重置階段不會自動激活，但**可以透過外部效果（如觸發 Active、站登）轉為激活**。
- **`pacify`**: **此角色不能攻擊及不能阻擋**。指定角色在期間內無法宣告攻擊，也無法宣告阻擋。
- **`stand_on_play`**: **以激活狀態在場上登場**。登場時若為休息狀態，則強制轉為激活狀態。
- **`swap_position`**: **對調**。選擇場上兩張角色卡交換位置。
- **`move_line`**: **移動站線**。將角色在前線與能量線之間切換。
- **`lockdown_movement`**: **不能移動**。指定角色無法在前後線之間移動。
- **`unblockable_by_bp`**: **此角色不會被BPxxxx或以/下的角色阻擋**。指定角色的攻擊無法被指定BP上或下阻擋。

## 5. 區域定義

- `front_line`: 戰鬥區 (最多 4 張)。
- `energy_line`: 能量區 (最多 4 張)。
- `hand`: 手牌 (最多 8 張)。
- `deck`: 牌庫(最多 50 張)。
- `trash`: 退場區。
- `outside`: 外置區。
